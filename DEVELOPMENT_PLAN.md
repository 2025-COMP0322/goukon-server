# KNU-CAMPUS 프로젝트 현황 분석 및 개발 계획

> **프로젝트 개요**: 경북대학교 학생들을 위한 과팅(미팅) 매칭 서비스
> - 학생들이 그룹을 만들어 다른 성별의 그룹과 매칭
> - 1:1 또는 3:3 매칭 지원
> - 매칭 후 실시간 채팅 가능

---

## 1. 현재 구현된 기능

### 완료된 기능
| 기능 | 설명 | 상태 |
|------|------|------|
| 회원가입 | 이메일, 닉네임, 비밀번호, MBTI, 성별 입력 | ✅ |
| 로그인 | 이메일/비밀번호로 로그인 | ✅ |

## 2. 스키마 수정 필요사항

### 결정사항: Member → Student 통합
> 현재 Member(인증용)와 Student(프로필용)가 분리되어 있으나, SQL 스키마에 맞춰 **Student 하나로 통합**

### 스키마 변경 내역

| 구분 | 변경 전 | 변경 후 | 비고 |
|------|---------|---------|------|
| **Student** | id, studentNumber, age, gender, name, mbti, profile, department | + email, password 추가 | Member 필드 흡수 |
| **Group** | id, gender | + status (AVAILABLE/QUEUING/MATCHED) 추가 | 그룹 상태 관리 |
| **MatchingSession** | id, type | + status (ACTIVE/COMPLETED/CANCELED), type 제거 | 세션 상태 관리 |
| **StudentGroup** | ❌ 없음 | 신규 생성 (student_id, group_id) | 학생-그룹 M:N 관계 |
| **MatchingQueue** | ❌ 없음 | 신규 생성 | 매칭 대기열 |
| **SessionMatches** | ❌ 없음 | 신규 생성 | 세션-그룹 연결 |
| **Report** | ❌ 없음 | 신규 생성 | 신고 기능 |
| **Member** | 존재함 | **삭제** | Student로 통합 |

## 3. 프론트엔드 페이지 기획 (사용자 관점)

### 🏠 메인/홈 화면
사용자가 앱에 들어오면 처음 보는 화면
- 서비스 소개 문구
- "로그인" / "회원가입" 버튼
- (로그인 시) 현재 매칭 상태 요약, 새 메시지 알림

---

### 🔐 회원가입/로그인

#### 회원가입 화면
1. **기본 정보 입력**
   - 이메일 (경북대 이메일 권장: @knu.ac.kr)
   - 비밀번호 / 비밀번호 확인
   - 이름
   - 학번

2. **프로필 정보 입력**
   - 나이
   - 성별 (남/여)
   - 학과
   - MBTI (선택)
   - 자기소개 (선택)

#### 로그인 화면
- 이메일/비밀번호 입력
- "로그인" 버튼
- "비밀번호 찾기" 링크
- "회원가입" 링크

---

### 👤 내 프로필

#### 프로필 조회/수정 화면
- 내 사진 (추후 구현)
- 이름, 학과, 나이, 성별, MBTI
- 자기소개
- "수정하기" 버튼 → 정보 수정 가능
- "비밀번호 변경" 버튼

---

### 👥 그룹 관리

#### 내 그룹 목록 화면
- 내가 속한 그룹 목록 표시
- 각 그룹 카드에:
  - 그룹 멤버들 (예: 홍길동, 김철수, 이영희)
  - 그룹 상태 (대기중/매칭중/매칭완료)
- "새 그룹 만들기" 버튼

#### 그룹 만들기 화면
- 매칭 유형 선택: "1:1 (혼자)" 또는 "3:3 (3명)"
- 1:1 선택 시: 바로 그룹 생성
- 3:3 선택 시:
  - 친구 검색 (학번 또는 이름으로)
  - 친구 초대 버튼
  - 현재 멤버 목록 (최대 3명)
  - "그룹 생성 완료" 버튼

#### 그룹 상세 화면
- 그룹 멤버 목록
- 각 멤버의 간단한 프로필 (이름, 학과, MBTI)
- 그룹장인 경우: "멤버 내보내기", "그룹 삭제" 버튼
- 멤버인 경우: "그룹 나가기" 버튼

---

### 💕 매칭

#### 매칭 신청 화면
- 매칭 신청할 그룹 선택
- "매칭 시작하기" 버튼
- 매칭 조건 설정 (추후):
  - 선호 나이대
  - 선호 학과
  - 선호 MBTI

#### 매칭 대기 화면
- "매칭 대기중..." 애니메이션
- 예상 대기 시간 (옵션)
- "매칭 취소하기" 버튼

#### 매칭 성공 화면
- 🎉 "매칭되었습니다!" 메시지
- 상대 그룹 멤버 프로필 미리보기
  - 이름, 학과, 나이, MBTI
- "채팅방 입장하기" 버튼

---

### 💬 채팅

#### 채팅방 목록 화면
- 내가 참여중인 채팅방 목록
- 각 채팅방에:
  - 상대 그룹 이름 또는 대표 멤버 이름
  - 마지막 메시지 미리보기
  - 읽지 않은 메시지 수 (빨간 뱃지)
  - 마지막 메시지 시간

#### 채팅방 화면
- 상단: 상대 그룹 정보 (멤버 목록 보기 버튼)
- 중앙: 메시지 목록
  - 내 메시지 (오른쪽)
  - 상대 메시지 (왼쪽, 발신자 이름 표시)
  - 시간 표시
- 하단: 메시지 입력창 + 전송 버튼
- 우측 상단: 신고하기 버튼 (⚠️)

---

### 🚨 신고

#### 신고하기 화면 (채팅방에서 접근)
- 신고 대상: 자동으로 상대 그룹 또는 특정 멤버 선택
- 신고 제목 입력
- 신고 내용 상세 입력
- 스크린샷 첨부 (추후)
- "신고 제출" 버튼

#### 내 신고 내역 화면
- 내가 접수한 신고 목록
- 각 신고:
  - 제목
  - 상태 (접수됨/검토중/처리완료/반려)
  - 접수 날짜

---

## 4. 백엔드 구현 필요 기능

### 학생 관련
- 학생 프로필 조회/수정
- 학생 검색 (친구 초대용)

### 그룹 관련
- 그룹 생성
- 그룹 조회 (내 그룹 목록)
- 그룹에 멤버 추가/제거
- 그룹 삭제

### 매칭 관련
- 매칭 대기열 등록
- 매칭 대기열 취소
- 매칭 상태 조회
- 자동 매칭 실행 (백그라운드)

### 채팅 관련
- 채팅방 목록 조회
- 채팅 메시지 조회 (과거 메시지)
- 실시간 메시지 전송 (이미 구현됨)

### 신고 관련
- 신고 접수
- 내 신고 목록 조회
- 신고 상태 변경 (관리자용)

## 5. 개발 우선순위

### 1단계: 스키마 정리
1. Member 엔티티를 Student로 통합
2. 누락된 엔티티 생성 (StudentGroup, MatchingQueue, SessionMatches, Report)
3. 기존 엔티티에 필드 추가 (status 등)

### 2단계: 핵심 API 구현
1. 학생 프로필 API
2. 그룹 CRUD API
3. 매칭 API
4. 신고 API

### 3단계: 프론트엔드 (미정)
- 프레임워크 결정 후 진행
